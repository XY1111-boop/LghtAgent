**标题**：RESTful API 核心方法与状态码

**关键概念**：

- **状态码**：200（成功）、404（未找到）、500（服务器内部错误）
- **RESTful API 设计**：基于资源的架构风格，使用标准 HTTP 方法进行 CRUD 操作

**摘要**：
本文档概述了 RESTful API 设计中常用的四种核心 HTTP 方法（GET、POST、PUT、DELETE），分别对应资源的读取、创建、更新和删除操作。同时列出了三个关键状态码：200 表示请求成功，404 表示资源未找到，500 表示服务器内部错误。这些元素共同构成了 RESTful API 的基本通信规范。
