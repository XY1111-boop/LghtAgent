**标题**：Docker 核心概念与指令解析

**关键概念**：
- **Dockerfile**：用于定义镜像构建步骤的脚本文件，包含基础镜像、依赖安装、文件复制、启动命令等指令。
- **docker-compose**：用于定义和运行多容器 Docker 应用的工具，通过 YAML 文件配置服务、网络、数据卷等。
- **镜像**：轻量级、可执行的独立软件包，包含运行应用所需的所有内容（代码、运行时、库、环境变量、配置文件）。
- **容器**：镜像的运行实例，具有隔离性、可移植性，可通过 Docker API 或 CLI 创建、启动、停止、删除。
- **数据卷**：用于持久化容器生成或使用的数据，独立于容器的生命周期，支持数据共享与备份。
- **CMD vs ENTRYPOINT**：
  - **CMD**：提供容器启动时的默认命令或参数，可被 `docker run` 后跟的命令覆盖。
  - **ENTRYPOINT**：将容器配置为可执行程序，启动时始终执行指定命令，可结合 CMD 提供默认参数。

**摘要**：
本文档介绍了 Docker 生态中的核心概念与关键指令。Dockerfile 用于构建自定义镜像，docker-compose 简化多容器编排。镜像与容器是 Docker 的基础单元，数据卷解决持久化存储问题。CMD 与 ENTRYPOINT 的区别在于：CMD 提供可被覆盖的默认命令，而 ENTRYPOINT 定义容器的主执行程序，两者可配合使用以实现灵活的参数传递。
