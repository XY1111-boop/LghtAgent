**标题**  
Git 基础操作与分支管理

**关键概念**  
- **git init**：初始化一个新的 Git 仓库。  
- **git add**：将工作区的更改添加到暂存区。  
- **git commit**：将暂存区的更改提交到本地仓库。  
- **git push**：将本地仓库的提交推送到远程仓库。  
- **git pull**：从远程仓库拉取最新更改并合并到本地。  
- **git branch**：创建、查看或删除分支。  
- **git merge**：将指定分支的更改合并到当前分支。  
- **解决冲突**：当合并时出现代码冲突，需手动编辑文件并重新提交。

**摘要**  
本文档介绍了 Git 的核心操作流程（init、add、commit、push、pull）以及分支管理（branch、merge），并强调了在合并过程中可能出现的冲突及其解决方法。这些命令是日常版本控制的基础，适用于个人开发与团队协作。
