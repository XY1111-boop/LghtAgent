### 📚 知识卡片：Web 漏洞演示（教育版）

**标题**：常见 Web 安全漏洞：SQL 注入与 XSS

**关键概念**：
- **SQL 注入**：通过构造恶意输入（如 `' OR '1'='1`）篡改 SQL 查询语句，绕过身份验证或获取未授权数据。
- **参数化查询**：将用户输入与 SQL 语句分离，防止恶意代码被执行，是防御 SQL 注入的有效方法。
- **ORM（对象关系映射）**：通过抽象数据库操作，自动处理参数绑定，降低注入风险。
- **XSS（跨站脚本攻击）**：在网页中注入恶意脚本，攻击者可通过未转义的输出执行脚本，窃取用户数据或篡改页面。
- **输出转义**：对用户输入的内容进行编码或过滤，防止浏览器将其解释为可执行脚本。

**摘要**：
本文档以教育为目的，演示了两种常见的 Web 安全漏洞：SQL 注入和跨站脚本攻击（XSS）。通过一个简单的登录查询示例，展示了攻击者如何利用未过滤的用户输入（如 `' OR '1'='1`）绕过身份验证。文档指出，防御 SQL 注入的关键在于使用参数化查询或 ORM 框架，避免直接拼接 SQL 语句。同时，针对 XSS 攻击，强调必须对输出内容进行转义处理，防止恶意脚本在用户浏览器中执行。这些措施是构建安全 Web 应用的基础实践。
