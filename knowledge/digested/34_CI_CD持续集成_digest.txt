**标题**：CI/CD 工具与配置概览

**关键概念**：
- **GitHub Actions**：基于 GitHub 仓库的持续集成与持续部署（CI/CD）平台，通过事件触发自动化工作流。
- **Jenkins**：开源的自动化服务器，支持构建、测试、部署等任务，可通过插件扩展功能。
- **自动构建、测试、部署**：软件开发流程中的核心自动化环节，旨在提高效率、减少人工错误。
- **YAML 配置**：一种人类可读的数据序列化格式，常用于定义 CI/CD 流水线的步骤与参数（如 GitHub Actions 的 `.github/workflows/*.yml` 文件）。

**摘要**：
本文档简要介绍了 GitHub Actions 和 Jenkins 两种主流的 CI/CD 工具，强调它们均支持自动构建、测试与部署流程，并采用 YAML 格式进行配置管理。
