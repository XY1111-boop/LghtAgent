### 📘 知识卡片：API 认证与安全基础

**标题**：API 认证机制与安全最佳实践

**关键概念**：
- **JWT (JSON Web Token)**：一种自包含的、基于 JSON 的令牌格式，用于在各方之间安全传输信息，常用于身份验证和信息交换。
- **OAuth2**：一个开放标准的授权框架，允许第三方应用获取对用户资源的有限访问权限，而无需暴露用户密码。
- **API Key**：一种简单的身份验证方式，通常是一个唯一字符串，用于识别调用 API 的客户端。
- **Bearer Token**：一种访问令牌，持有该令牌即可访问受保护的资源，通常用于 OAuth2 或 JWT 认证中。

**摘要**：
本文档介绍了四种常见的 API 认证与授权机制：JWT、OAuth2、API Key 和 Bearer Token。JWT 是一种轻量级的令牌格式，常用于无状态认证；OAuth2 是一个授权框架，适用于第三方应用授权场景；API Key 是最简单的身份识别方式，但安全性较低；Bearer Token 是一种常见的令牌传递方式，通常与 JWT 或 OAuth2 结合使用。文档还强调了在实现这些机制时应遵循的安全最佳实践，例如使用 HTTPS、避免硬编码密钥、定期轮换令牌等。
